# Frontend Continuation Guide

Status date: 2026-06-06

`packages/frontend` is the product frontend package. Do not create a new frontend tree for HOPI.

This package should continue from its original React/Vite codebase:

- `src/App.tsx`
- `src/components/Layout.tsx`
- `src/pages/BoardView.tsx`
- `src/pages/SessionView.tsx`
- `src/components/AssistantPanel.tsx`

The backend-served UI under `packages/backend/src/ui` is a capability and API reference surface only. It should help answer “what product surfaces exist?” and “which backend routes/state already exist?”, but it should not be copied wholesale as the frontend structure or visual language.

## Current Status

The original frontend baseline has been restored and the package now verifies again:

```sh
cd packages/frontend
bun run check
```

That means:

- `bun run typecheck` passes
- `bun run build` passes

It does **not** mean the frontend has full parity with the currently implemented backend product surfaces.

## Frontend Rule

When adding product capability to the frontend:

1. Keep the existing React/Vite tree as the base.
2. Evolve or replace individual pages/components inside `packages/frontend`.
3. Use backend UI only as a capability checklist or API-reference aid.
4. Do not transplant backend HTML/CSS/DOM structure into this package as the primary approach.
